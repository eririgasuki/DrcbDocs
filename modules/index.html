# 页面功能

C端在一个公共页面中，通过调用外设模块和常用库/插件，去实现一个公共的功能模块。

同时根据不同交易进行相应处理，满足不同的交易需求。

//外设读介质
let result = await PassbookPrinter.readTracksAsync();
let trackNum = result.Track3
//分交易做处理
if (
  this.BUSISTEP === "GetTOutBankBook_2" &&
  trackNum.substring(14, 22) === "01561560"
) {
  ExceptionHelper.pushError("对私交易不能插入对公介质");
} else if (
  this.BUSISTEP === "GetTCorpBankBook_2"&&
  trackNum.substring(0, 6) === "940062"
) {
  ExceptionHelper.pushError("对公交易不能插入对私介质");
}

# 外设模块

在应用中，C端使用 js SDK 通过 cordova 插件，调用 C# 中间件，C# 中间件调用硬件厂商 SP 服务，SP 服务去操作硬件设备。

通常，C端需要使用文字、图片、语音、指示灯等方法提示用户操作。同时调用外设去吸入、读取、退出介质（身份证，银行卡，存折，票据等），或从卡箱，钞箱，UKEY箱中发出物料（新开银行卡、UKEY，取款），或使用高拍仪获取影像，打印机打印文档等。

同时，通过 await 外设操作完成，或监听（onItemsInserted, onItemsPresent, onItemsTaken 等）事件触发去控制硬件设备的调用流程。

async readIDCardAsync() {
  this.pageData.title = "请插入您的二代居民身份证";
  AudioPlayer.play("insertIDCard");
  SensorsIndicators.openIDCardGuidLightAsync();
  try {
    this.deviceData.IDData = await IDCardReader.readIdCardAsync();
  } catch (error) {
    console.error("设备读取身份证失败:", error);
    TradeLogger.error("设备读取身份证失败:", error);
    ExceptionHelper.pushError("设备读取身份证失败");
  }
  console.log("身份证数据", this.deviceData.IDData);
}

目前C端使用的外设模块如下：

  • 身份证模块 IDCardReader
  • 银行卡模块 CardReader CardDispenser
  • 存折模块 PassbookPrinter
  • 密码模块 PinKeypad
  • 高拍仪模块 HighCamera
  • 文档模块 DocumentReader DocumentPrinter
  • 票据模块 DepositReader DepositPrinter
  • 凭条模块 ReceiptPrinter
  • 现金存款模块 CashIn
  • 现金取款模块 CashDispenser
  • U盾模块 UKeyReader UKeyDispenser

  • 签字版模块 SignatureTablet
