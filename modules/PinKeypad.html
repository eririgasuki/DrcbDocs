父组件 CheckPinkeyPad 通过调用 pinKeyPad 组件的 startAsync 方法，开启密码的获取流程

<pin-keypad ref="pinKeyPad" :cardNumber="cardNumber" @passwordgotton="GetPassword" />

onNavigated() {
  context.REQ_BODY.SHOWINIT.map(val=>{ //用来给密码组件加密的PASSWD_ACCNO字段在SHOWINIT而不是通常的SHOWDATA下。
    val.FIELDCODE == "PASSWD_ACCNO" && (cardNumber = val.FIELDVALUE) //CheckPinkeyPad父组件获取卡号传值给pinKeyPad子组件
  })
  this.$refs.pinKeyPad.startAsync();
},

以下是 pinKeyPad 子组件中的流程

async startAsync() {
  PinKeypad.addOnKeyPressedEventListener( //监听键盘事件
    this.$options.name,
    this.onKeyPressedEvent
  );
  await PinKeypad.getPinAsync({
    TerminateKeys: 0,
    TerminateFDKs: 0,
    ActiveKeys: 12287, // 数字键+clear
    ActiveFDKs: 0,
    Echo: 0,
    AutoEnd: true,
    MaxLen: 6,
    MinLen: 6
  }); //等待用户输完6位密码
  let pinBlock = await PinKeypad.getPinBlockExAsync({ //根据父组件传的卡号cardNumber加密
    Algorithm: 4096,
    KeyEncKey: null,
    Key: "PinKeySM4",
    Format: 2,
    Padding: 0,
    XORData: null,
    CustomerData: cardNumber
  });
  this.$emit("passwordgotton", result); //向父组件CheckPinkeyPad传递passwordgotton事件的值，即加密后的密码pinBlock
  PinKeypad.removeOnKeyPressedEventListener( //取消键盘事件的监听
    this.$options.name,
    this.onKeyPressedEvent
  );
},

通过监听键盘事件 onKeyPressedEvent 更新页面中的密码UI组件

<aui-password ref="pwInput"></aui-password>

onKeyPressedEvent(args) {
  if (this.keyCount >= 6) { //输完6个键后，不再更新UI
    return;
  }
  if (args.Key == "0") { //数值事件
    this.keyCount++;
    this.$refs.pwInput.addPsw();
  } else if (args.Key == "8192") { //回退事件
    this.keyCount--;
    this.$refs.pwInput.backPsw();
  } else if (args.Key == "2048") { //清空事件
    this.keyCount = 0;
    this.$refs.pwInput.clearPsw();
    await PinKeypad.cancelAsync(); //清空后取消PinKeypad.getPinAsync方法的等待
    this.startAsync(); //重新开始获取密码流程
  }
},

pinkeypad组件是和智能柜台共用的，如有改动一定要知会智能柜台一起讨论。

业务提到过物理键盘上的确定键提交密码的需求：

  1. 智能柜台是输完6位密码后passwordgotton事件触发时直接提交。
  2. 而测试之处业务定的云柜台不自动提交，需用户手动点提交。（包括设置密码和短信验证码）
  3. 生产上的业务提出过物理键盘不能提交，体验不好。
  4. 目前pinkeypad组件不能实现，从startAsync流程中可以看到：
  5. ActiveKeys是不包含确认键
  6. 输完6位密码后就取消监听键盘事件了

如果业务明确要实现物理键盘确定可以提交。 方向一：

  • 另修改出一个pinkeypad组件，避免影响智能柜台。
  • 确认ActiveKeys支持确认键。
  • 新增确认键事件的监听。
  • pinkeypad组件新增一个确认事件，keyCount为6且按确定键后用于向父组件提交。
  • 取消监听事件从获取密码后，改到beforeDestroy时执行。

方向二：

  • 密码键盘的确定按钮和普通键盘的回车键会有同样的效果。
  • 当输完6位密码后passwordgotton事件触发时，聚焦提交按钮this.$refs.submitBtn.$el.onfocus()。
  • 用户按密码键盘确认按钮后，就会触发提交按钮的click事件。
  • 目前不是所有机具都支持，需要厂商统一改动SP。

titleMap = {
  SetPwd_1: "请设置您的手机银行密码",
  PRI_PerlCompreSign_SetEbankPayPwd_1: "请设置您的网银支付密码",
  PRI_PerlCompreSign_SetEbankLoginPwd_1: "请设置您的网银登录密码"
}
title = flowID === "PerlCompreSign" ? titleMap[this.BUSISTEP] : "请设置您的密码";

校验设置的密码是否过于简单的接口TC046的入参VOUNO字段P端没送，又不能为空。故从PASSWD_ACCNO字段截取。

cardNumber = this.find("PASSWD_ACCNO") || "0000000000000"
const request = new TC046();
request.VOUNO = cardNumber.slice(-13);
