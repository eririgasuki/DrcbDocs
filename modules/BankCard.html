# 注意点&可能的扩展改造

只有BUSISTEP === ReadPerUnitBankCard_1才有当前坐席选择的对公、对私字段 CSTMARK (1-对私 2-对公)。

  • 读取到的银行卡号CardNumber通常由6位卡bin和13位凭证号组成。
  • 识别卡除外，通常由45开头的13位凭证号和结尾补位的888888组成。
  • 目前卡号都是直接通过19位长度截取处理，如果新增卡号长度改变，相应截取规则也需跟进改变。
if (
  //识别卡
  CardNumber.slice(0, 2) === "45" &&
  CardNumber.slice(-6) === "888888"
) {
  accNo = CardNumber.slice(0, 13);
  vouNo = CardNumber.slice(0, 13);
} else {
  accNo = CardNumber;
  vouNo = CardNumber.slice(-13);
}

凭证号前两位可以确定银行卡类型，映射关系建议改成接口获取。

//私人银行卡: 1; 单位结算卡: a; 识别卡: 4
mediaTypeMap = {
  "61": "a",
  "45": "4",
  "28": "1",
  "88": "1"
};

识别卡也分对私（1）和对公（2），通过调接口返回CLIENT_NO字段的首位判断

const request = new TQ105();
request.ACCT_NO = CardNumber.slice(0, 13);
result = await RequestHelper.sendAsync(request);
discernCardType = this.find("CLIENT_NO", result).slice(0, 1)

是否是他行卡是写在本地的卡bin数组判断的，建议改成接口获取。

LegalbinList = ["622328", "623038", "625288", "625888", "623198", "940062", "623265", "628298"]
!LegalbinList.includes(CardNumber.slice(0, 6)) && (isOtherCard = true);

校验卡号是否符合交易规则，是根据 BUSISTEP 划分的。

checkMap = {
  ReadTOutBankCard_1: isUnitSettleCard || isOtherCard,
  ReadTOutDiscernCard_1: isUnitSettleCard || isOtherCard,
  ReadUnitSettleCard_1: isPersonalCard || isOtherCard,
  ReadOtherBankCard_1: isUnitSettleCard || !isOtherCard,
  ReadCreditCard_1: isOtherCard
};
checkMap[this.BUSISTEP]
  ? ExceptionHelper.pushError("卡号不符合当前交易规则") && (await this.ejectCard())
  : this.sendAsync();

# 外设方法注意点

信用卡一定是芯片卡，读取信用卡（贷记卡）和储蓄卡（借记卡）的方法不一样。

this.BUSISTEP === "ReadCreditCard_1"
  ? await CardReader.pbocGetAppInfoByCreditAIDAsync() // 通过贷记卡 ID 获取卡信息
  : await CardReader.pbocGetAppInfoByDebitAIDAsync(); // 通过借记卡 ID 获取卡信息

读芯片卡的方法有顺序要求(举例借记卡)

await CardReader.readCardChipAsync(); // 读取芯片卡（吸入动作）
await CardReader.powerUpAsync(); //上电
await CardReader.pbocGetAppInfoByDebitAIDAsync(); // 通过借记卡 ID 获取卡信息
await CardReader.pbocAppInitializeByParamsAsync('000000000000', '00'); //初始化应用
cardData = await CardReader.pbocReadAppDataAsync(); //：读取卡片数据

只有芯片卡才有55域数据,需要在读取芯片卡数据后读取55域数据

await CardReader.pbocStartTradeAsync(); //开始交易
field55 = await CardReader.pbocGenerateUnionpayField55ByRequiredFieldAsync(); //获取银联交易中 55 域信息

因为cancelAsync可能会导致卡顿，所以未插入介质时关闭页面后，为了避免插入介质后函数继续执行，会使用return出去。

async onClose() {
  // CardReader.cancelAsync();
  !this.mediaInserted && (this.needReturn = true);
},

async onNavigated() {
  await CardReader.readCardTrack23Async(); //未插入介质关闭页面时，读取方法仍然在等待中
  if (this.needReturn) {
    return;
  }
  this.mediaInserted = true;
}
