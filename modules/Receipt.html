# PrintSlip 凭条打印页面

# 简易流程图

# 注意点&可能的扩展改造

打印出的凭条小票如图：

凭条小票分为交割小票和普通小票，两者公共凭条要素略有不同。 交割小票：可供客户去柜面领取/交付物料(凭证或现金)。 普通小票：仅供客户核对之用，不作任何凭证。

凭条打印内容由A端推送，为放在 PRINTDATA 字段下的数组里，里面每个数组代表每笔交易的凭条内容。

{
  "FIELDCODE": "PRINTDATA",
  "FIELDVALUE": [
    [
      "交易流水：81202005286121L00001",
      "交易时间：10:16:57",
      "交易名称：个人综合签约",
      "账号：1200100062****5",
      "账户名称：钟*科",
      "交易种类：个人手机银行新增",
      "签约类型：签约版",
      "手机号码：138****2200",
      "交易结果：成功",
    ],
    [
      "交易流水：81202005286121L00018",
      "交易时间：14:52:17",
      "交易名称：个人综合签约",
      "账号：1200100062****5",
      "账户名称：钟*科",
      "交易名称：个人网上银行新增",
      "签约类型：专业版",
      "证书下载校验码：12001202005280027381(请于14个自\\L然日内下载证书，否则校验码会失效)",
      "USBKEY编号：48**********28",
      "手机号码：138****2200",
      "交易结果：成功"
    ]
  ]
}

let printDataList = this.find("PRINTDATA");
for (let i = 0; i < printDataList.length; i++) {
  this.PrintData.push(""); //添加空行
  printDataList[i].forEach(element => {
    this.PrintData.push(element);
  });
}

最终实际传入打印小票的参数是拼接后的字符串，"\\H"代表标题（样式加粗居中字体加大）,"\\L"代表普通内容。

ReceiptPrinter.printAsync("\\H东莞农村商业银行客户凭条\\L交易流水：81202005286121L00001\\L交易时间：10:16:57\\L\\L交易名称：周周到新增\\L预留金额：￥8,888.00")

凭条打印机在纸少的时候，也可能会打印失败。具体多少算纸少是按厂家硬件设计来判断的。

凭条纸背面黑线是切割的位置，在小票长度超过一张时，部分厂商曾会打印失败报错（厂商已修复）。
