# 简述

  • 如 Router 中提到，PageContainerNavigator 已经监听处理 openPage 事件的信息作为 context 传递给云柜台交易页面子组件。
  • 同时 BasicPage 已混入该 props ，所以交易页面中可以直接使用 this.context 取到报文的值。

# 报文事件

  • 在 events 中，已监听 ReverseCommandReceived 事件，触发时执行 RemoteCommandManager.invokeCommand 方法。
  • RemoteCommandManager 根据指令类型 CommandType 发布订阅消息，报文的指令类型为 OpenPage 。
  • 报文 this.context 获得的内容为 eventData.EventParameters.CommandParameters 。

# 报文内容

  • 如 Router 中提到，云柜台交易页面路由配置的 BUSISTEP 即报文 this.context.REQ_BODY.BUSISTEP 的值。
  • 云柜台交易页面中所需要用到的数据，通常放在 this.context.REQ_BODY.SHOWDATA 中，SHOWDATA 是一个数组，数据以 { FIELDCODE:"XXX", FIELDVALUE:"YYY" } 的对象形式放在其中。
  • 通常可以使用数组的 find 方法，通过 this.context.REQ_BODY.SHOWDATA.find(e=>e.FIELDCODE === "XXX").FIELDVALUE 快速取到所需要的值。
  • 也可以使用数组的 map 方法，通过 this.context.REQ_BODY.SHOWDATA.map((val, index, array)=>{if(val.FIELDCODE === XXX){this.xxx = val.FIELDVALUE}} 取到所需要的值。
